# Security Policy

## Supported versions

`meigma/packages` is an internal repository-automation tool without published
binary releases. Security fixes are applied to the latest code on `main`; older
commits are not maintained as supported release lines.

## Reporting a vulnerability

Report vulnerabilities through GitHub private vulnerability reporting for
`meigma/packages`.

Do not use public GitHub issues, pull requests, discussions, chat channels, or
other public forums for vulnerability reports.

Include the affected commit, impact, reproduction steps, and any suggested
mitigation when available. Avoid attaching live credentials, signing material,
or other secrets to the report.
